Условные обозначения:
(ниббл  это единица информации, равная четыре бита или половина байта)

    X: Вторая тетрада (4 бита) — обычно индекс регистра Vx.

    Y: Третья тетрада — обычно индекс регистра Vy.

    N: Четвертая тетрада — 4-битное число.

    NN: Последние 8 бит — 8-битное число.

    NNN: Последние 12 бит — адрес памяти.

Опкод	Описание
00E0	Очистка экрана (CLS).
00EE	Возврат из подпрограммы (RETURN).
1NNN	Переход по адресу NNN (JP addr).
2NNN	Вызов подпрограммы по адресу NNN (CALL addr).
3XNN	Пропуск следующей инструкции, если VX == NN (SE Vx, byte).
4XNN	Пропуск следующей инструкции, если VX != NN (SNE Vx, byte).
5XY0	Пропуск следующей инструкции, если VX == VY (SE Vx, Vy).

8XY1	Установить VX = VX OR VY.
8XY2	Установить VX = VX AND VY.
8XY3	Установить VX = VX XOR VY.
8XY4	Установить VX = VX + VY, флаг VF = 1, если есть перенос (carry).
8XY5	Установить VX = VX - VY, флаг VF = 0, если был заем (borrow).
8XY6	Сдвиг VX вправо на 1 бит. VF = отброшенный бит.
8XY7	Установить VX = VY - VX, флаг VF = 0, если был заем.
8XYE	Сдвиг VX влево на 1 бит. VF = отброшенный бит.

ANNN	Установить адресный регистр I = NNN (LD I, addr).
BNNN	Переход по адресу NNN + V0 (JP V0, addr).
CXNN	Установить VX = random_byte AND NN (RND Vx, byte).
DXYN	Рисование спрайта (DRW Vx, Vy, nibble). Координаты (VX, VY), высота N.

FX18	Установить звуковой таймер = VX.

FX29	Установить I на адрес спрайта символа, хранящегося в VX.

FX65	Заполнить регистры от V0 до VX из памяти, начиная с адреса I.

Итог:

    Массив — это оперативная память.
    NNN — это конкретный «номер ячейки» (индекс) в этом массиве.
    Регистр I — это «закладка» в памяти, которую вы перемещаете командами ANNN или FX1E, чтобы знать, откуда читать спрайты или куда записывать данные.

    Основные Компоненты CHIP-8:

        Память (4KB): Общий объем памяти, где хранится код программы, спрайты и данные.
        Регистры (V0-VF): 16 x 8-битных регистров для временного хранения данных.
        Индексный Регистр (I): 16-битный регистр для адресации памяти (например, для загрузки спрайтов).
        Стек: 8-битный стек для хранения адресов возврата, обычно 16 уровней.
        Счетчик Команд (PC): 16-битный указатель на следующую инструкцию.
        Таймеры: Таймер задержки (Delay Timer) и звуковой таймер (Sound Timer).
        Графика: Буфер кадра 64x32 пикселя для отрисовки спрайтов.

    Принцип Работы (Цикл Эмуляции):

        Цикл Fetch-Decode-Execute: Интерпретатор CHIP-8 непрерывно выполняет цикл:
            Fetch (Получение): Читает 2 байта (одну инструкцию) из памяти по адресу, указанному в PC.
            Decode (Декодирование): Определяет, что делать, на основе полученного 16-битного кода операции.
            Execute (Выполнение): Выполняет действие (например, сложить регистры, переместить данные, нарисовать пиксель).
        Выполнение Инструкций: Каждая из 35 команд выполняет определённую функцию, например:
            0x00E0: Очистить экран.
            0x1NNN: Перейти к адресу NNN.
            0x6XNN: Сохранить значение NN в регистр VX.
            0x8XY0: Перенести значение из VY в VX.
            0xDXYN: Нарисовать спрайт.
        Обновление Состояния: После выполнения инструкции PC сдвигается на 2 байта, таймеры обновляются, и цикл повторяется
